# Building & Development — Kite Ground Control

This document describes how to set up the development environment and build the application.

**Recommended tool**: We use **[just](https://github.com/casey/just)** as the primary task runner. It provides a clean, consistent interface across Windows and Linux.

---

## Prerequisites

**Important:** All heavy system dependencies listed below must be installed **manually** by the developer. There are no automated scripts for these, as they require significant system changes and often administrative rights.

### All Platforms
- **Node.js** LTS (v20 or v24 recommended)
- **Rust** (installed via [rustup](https://rustup.rs/))
- **[just](https://github.com/casey/just)** (task runner) — strongly recommended for development

### Windows (Primary Development Platform)

You need the following components:

1. **Visual Studio Build Tools 2022** (or full Visual Studio 2022)  
   Required workload: **"Desktop development with C++"**

   This installs the MSVC compiler and linker that Rust needs.

2. **WebView2 Runtime**  
   Usually already installed on Windows 10/11. If missing, it can be downloaded from Microsoft.

**Recommended installation (via winget):**
```powershell
winget install OpenJS.NodeJS.LTS
winget install Casey.Just.Just
```

Rust is best installed from the official site: https://rustup.rs/

> **Important:** After installing any of the above tools, **completely restart your terminal and VSCode** so the new paths are recognized.

### Linux (Debian / Ubuntu based)

You need the following system packages:

```bash
sudo apt update
sudo apt install -y \
    build-essential \
    curl \
    wget \
    file \
    libssl-dev \
    libgtk-3-dev \
    libwebkit2gtk-4.1-dev \
    libayatana-appindicator3-dev \
    librsvg2-dev
```

These packages are required by Tauri to build the Linux desktop application.

**Node.js, Rust, and just** should be installed via their official methods (not necessarily via apt).

### Android Development

Android support via Tauri Mobile was partially experimented with but is currently **on hold**.

It requires a completely separate UI and build pipeline. Therefore, Android-related setup is **not documented** here and should not be attempted until this is officially resumed.

Do not run `tauri android init` or similar commands at this time.

## Recommended Workflow (using `just`)

After cloning the repository and installing the prerequisites:

```powershell
# 1. Install frontend dependencies
just install

# 2. Start development mode (hot reload)
just dev
```

Other useful commands:

```powershell
just --list          # Show all available commands
just check           # Run svelte-check + cargo check
just build           # Production build for current platform
just build-windows   # Explicit Windows release build
just build-linux     # Explicit Linux release build
just clean           # Clean build artifacts
```

## Running Quality Checks

This project relies heavily on static analysis because large parts were generated by AI.

```powershell
just check
```

This runs:
- `npm run check` (svelte-check + TypeScript)
- `cargo check` (in `src-tauri/`)

We also have a CI workflow that executes these checks on every push and pull request.

## Common Issues & Troubleshooting

### "just is not recognized"
- Restart your terminal / VSCode completely.
- Verify installation with `where.exe just` in PowerShell.
- Make sure you installed it for the **current user** (winget usually does this correctly).

### VSCode Integrated Terminal doesn't find just / cargo / node
This is extremely common on Windows.
**Solution**: Fully close VSCode and reopen it (or reload the window with `Ctrl+Shift+P` → "Developer: Reload Window").

### OneDrive path issues (Windows)
Cargo builds can fail or become very slow when the project lives inside OneDrive (because of spaces and sync locks).

The project already works around this via `src-tauri/.cargo/config.toml`, which redirects the cargo target directory to `D:\cargo-target\kite-gc`.

Do **not** delete or modify this file unless you know what you're doing.

### Missing MSVC linker on Windows
If you get linker errors during `cargo check` or build, you are missing the C++ build tools.

Install "Desktop development with C++" via the Visual Studio Installer.

### File not found during Blackbox import or other native features
Make sure you are running a **debug build** (`just dev`) or a **release build** (`just build`). Some features (like external binaries) behave differently in dev vs release.

## Extending the Build System

All tasks are defined in the `justfile` in the project root.

To add a new command (example):

```just
# In justfile
format:
    cargo fmt --manifest-path src-tauri/Cargo.toml
    npm run format   # (if you add a frontend formatter later)
```

Then it will automatically appear when running `just --list`.

## Continuous Integration (CI)

A GitHub Actions workflow (`.github/workflows/ci.yml`) runs on every push and pull request. It currently executes:

- `npm run check`
- `cargo check`
- `clippy` (with warnings as errors on Linux)

Full release builds are **not** run in CI (they are slow and require platform-specific signing setup).
